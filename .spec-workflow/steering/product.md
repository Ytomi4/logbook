# Product Overview

## Product Purpose

Logbook は、**「何を読んだか」だけでなく「どう読んだか」を残せる**読書記録アプリです。

読書中に感じたこと、印象に残った単語、後から振り返りたい一文を「ログ」として記録し、タイムライン形式で振り返ることができます。

**解決する課題:**
- 読書中の気づきや感想を忘れてしまう
- 読んだ本の内容を後から思い出せない
- 読書記録が散在して管理が難しい

## Target Users

**主要ユーザー:**
- 読書が好きで、読書記録を残したい個人ユーザー
- 読書メモや引用を整理して管理したい人
- 読書の振り返りを習慣化したい人

**ユーザーのニーズ:**
- 読んでいる途中でも手軽にメモを残せる
- 気に入った引用を保存できる
- 時系列で読書記録を振り返れる
- 本の情報を簡単に登録できる

## Key Features

### Core Features

| 機能 | 説明 |
|------|------|
| タイムライン形式 | 読んでいる途中の記録も振り返れる。同じ本のログはグループ化 |
| ログの記録 | 印象に残った単語、後から振り返りたい一文を簡単に残せる（メモ/引用の区別） |
| 簡単な登録 | 国立国会図書館検索で書影付きの本を登録 |

### Supporting Features

- **キーボードショートカット**: `Alt + H`（タイムライン）、`Alt + B`（本一覧）、`Alt + N`（本登録）
- **クイック記録**: モーダルから素早くログを追加
- **ユーザー認証**: GitHub / Google ソーシャルログイン
- **公開プロフィール**: ユーザー名・アバター設定、タイムラインの共有（`/u/:username`）

## Product Principles

1. **シンプルさ優先**
   - 機能を絞り、直感的な UI を維持する
   - 複雑な設定は避ける

2. **高速な操作**
   - キーボードショートカットやクイック入力で、思考を中断せずに記録できる

3. **読書中に使える**
   - 読み終わってからではなく、読んでいる途中で記録することを重視

4. **データの永続性**
   - ユーザーの読書記録を安全に保存し、いつでもアクセス可能にする

## Business Objectives

- 読書習慣の継続をサポートする使いやすいツールの提供
- 個人の読書体験を価値ある記録として残せる環境の構築
- シンプルで軽量なアプリケーションの維持

## Success Metrics

| 指標 | 目標 |
|------|------|
| 継続利用 | ユーザーが継続的にログを記録している |
| ログ登録数 | 1ユーザーあたり月間10件以上 |
| 応答性 | ページロード 1秒以内 |

## Future Vision

### 短期（検討中）
- 検索・フィルタ機能（キーワードでログを検索）
- 読書統計（月間読書冊数、ログ数の可視化）

### 中長期（アイデア段階）
- 本の登録の簡便化（バーコードスキャン）
- 引用の簡便化（カメラからの OCR）
- エクスポート機能（Markdown / CSV）
- タグ・カテゴリによる分類
- 読書ステータス（読書中・読了・積読）
- オフライン対応（PWA 化）
