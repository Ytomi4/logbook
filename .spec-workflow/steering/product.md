# Product Overview

## Product Purpose

Logbook は、読書体験を記録・振り返るための Web アプリケーションです。読書中に感じたこと、印象に残った引用、メモをタイムライン形式で記録し、後から振り返ることができます。

**解決する課題:**
- 読書中の気づきや感想を忘れてしまう
- 読んだ本の内容を後から思い出せない
- 読書記録が散在して管理が難しい

## Target Users

**主要ユーザー:**
- 読書が好きで、読書記録を残したい個人ユーザー
- 読書メモや引用を整理して管理したい人
- 読書の振り返りを習慣化したい人

**ユーザーのニーズ:**
- 手軽に読書メモを残せる
- 気に入った引用を保存できる
- 時系列で読書記録を振り返れる
- 本の情報を簡単に登録できる（国会図書館検索連携）

## Key Features

1. **タイムライン表示**: 読書ログを時系列で一覧表示。同じ本のログが連続している場合は本ごとにグループ化され、読書の軌跡を振り返れる
2. **メモ・引用の記録**: 読書中の気づき（メモ）と印象的なフレーズ（引用）を区別して記録
3. **本の管理**: 本の登録・編集・削除。国立国会図書館 API との連携で書誌情報を自動取得
4. **キーボードショートカット**: `Alt + H`（タイムライン）、`Alt + B`（本一覧）、`Alt + N`（本登録）で素早く操作
5. **クイック記録**: モーダルから素早くログを追加

## Business Objectives

- 読書習慣の継続をサポートする使いやすいツールの提供
- 個人の読書体験を価値ある記録として残せる環境の構築
- シンプルで軽量なアプリケーションの維持

## Success Metrics

- **日次アクティブ利用**: ユーザーが継続的にログを記録しているか
- **ログ登録数**: 1ユーザーあたりの月間ログ数
- **アプリ応答性**: ページロード時間 1秒以内

## Product Principles

1. **シンプルさ優先**: 機能を絞り、直感的な UI を維持する。複雑な設定は避ける
2. **高速な操作**: キーボードショートカットやクイック入力で、思考を中断せずに記録できる
3. **データの永続性**: ユーザーの読書記録を安全に保存し、いつでもアクセス可能にする
4. **オフライン対応検討**: エッジコンピューティング（Cloudflare Workers）を活用し、低遅延で動作

## Monitoring & Visibility

- **Dashboard Type**: Web ベースの SPA（Single Page Application）
- **Real-time Updates**: 即時反映（ログ追加後にタイムラインが更新）
- **Key Metrics Displayed**: タイムライン、本ごとのログ数、日付グループ
- **Sharing Capabilities**: 現時点では未実装（将来的に共有機能を検討）

## Future Vision

### Potential Enhancements

- **デザインのブラッシュアップ**: UI/UX の改善、視認性・操作性の向上
- **本の登録の簡便化**: 検索機能の強化、バーコードからの登録など
- **引用の簡便化**: カメラからのテキスト読み取り（OCR）
- **検索・フィルタ機能**: キーワードやタグでログを検索
- **読書統計**: 月間読書冊数、ログ数の可視化
- **エクスポート機能**: Markdown や CSV でデータをエクスポート
- **タグ・カテゴリ**: 本やログにタグを付けて分類
- **読書ステータス**: 読書中・読了・積読などのステータス管理
- **複数ユーザー対応**: ユーザー認証と個人データの分離
